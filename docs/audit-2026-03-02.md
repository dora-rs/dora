## 2. Security Findings

### CRITICAL

**[S-1] Command injection via shell nodes** — `binaries/daemon/src/spawn/command.rs:21-43`
When `ADORA_ALLOW_SHELL_NODES=true`, raw YAML args are passed to `sh -c` without sanitization. An attacker who can supply a malicious YAML achieves full OS command execution.

**[S-2] Node args split naively on whitespace** — `binaries/daemon/src/spawn/command.rs:124`
`args.split_ascii_whitespace()` doesn't handle quoting, escaping, or null bytes. Migrate to `Vec<String>` in the YAML schema or use `shlex::split`.

### HIGH

**[S-3] Shared memory TOCTOU** — `libraries/shared-memory-server/src/channel.rs:173-188`
`msg_len` is read from shared memory controlled by the node. A compromised node can race between the bounds check and `slice::from_raw_parts`. Additionally, `bincode` v1 is vulnerable to allocation-bomb attacks.

**[S-4] Node-to-daemon TCP unauthenticated** — `binaries/daemon/src/node_communication/tcp.rs`
Any local process can connect to the daemon's TCP listener and register as a node. No per-node secret is verified.

**[S-5] Token accepted in URL query parameter** — `binaries/coordinator/src/ws_server.rs:40-49`
`?token=<hex>` fallback leaks tokens in access logs and referrer headers.

**[S-6] Shmem permissions TOCTOU** — `binaries/daemon/src/node_communication/mod.rs:100-121`
Shared memory under `/dev/shm` is briefly world-readable between creation and `set_permissions`. macOS has no hardening at all.

**[S-7] Git rev-spec injection** — `libraries/core/src/build/git.rs:357-373`
`commit_hash` from YAML is passed to `revparse_ext` which accepts arbitrary rev syntax. Should validate as `^[0-9a-fA-F]{40}$`.

## 3. Code Quality Findings

### CRITICAL (can crash daemon in production)

**[Q-1] `unwrap()` after map insert in hot data path** — `daemon/src/lib.rs:2141`
```rust
dataflow.publishers.insert(output_id.clone(), publisher);
dataflow.publishers.get(output_id).unwrap()
```
Use `entry().or_insert()` instead.

**[Q-2] `expect()` on log file creation and pipe handles** — `daemon/src/spawn/prepared.rs:413-415`
Disk-full or resource exhaustion crashes the entire daemon. Propagate as `eyre::Result`.

**[Q-3] `panic!` in NodeId/DataId constructors** — `libraries/message/src/id.rs:59,131`
`From<&str>` impl panics on invalid input. Any malformed ID from network/YAML panics the process.

## 5. Positive Findings

- Clippy and formatting are clean across entire workspace
- Auth token security: constant-time comparison, debug redaction (`AuthToken(***)`), CSPRNG via `getrandom`
- Resource bounds: named constants for all limits (`MAX_WS_CONNECTIONS`, `MAX_CONTROL_MESSAGE_BYTES`, etc.)
- ENV denylist blocks `LD_PRELOAD`, `DYLD_INSERT_LIBRARIES`, and auth token from spawned nodes
- HLC clock for distributed causality tracking
- Backpressure via `try_send` with dropped-frame counter for topic data
- Path traversal protection: `sanitize_node_id` blocks `..` and `/` with tests
- ROS2 name validation with strict character allowlists
- Unix domain socket directory permissions set to `0o700`

## 6. Recommended Priority

### Immediate (before next release)
1. Upgrade `pyo3` to >=0.24.1 (buffer overflow CVE)
2. Upgrade `protobuf` via prometheus update (recursion crash CVE)
3. Fix Q-1/Q-2: unwrap/expect in daemon hot paths
4. Gate `start_testing()` with `#[cfg(test)]` (Q-6)
5. Validate git commit hashes as hex (S-7)

### Short-term
6. Add per-node auth token for daemon TCP (S-4)
7. Remove `?token=` query param fallback (S-5)
8. Set umask before shmem creation (S-6)
9. Replace `subsec_nanos` jitter with PRNG (S-12)
10. Upgrade to `bincode` v2 with size limits (S-3)

### Medium-term
11. Split `daemon/src/lib.rs` monolith (Q-4)
12. Introduce context structs for 8-10 param functions (Q-5)
13. Add artifact store cleanup (Q-12)
14. Add per-IP WS rate limiting (S-9)
15. Reduce `unwrap()` count (369 calls in production)
