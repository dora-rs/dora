## 1. Security Findings

### CRITICAL

**[C1] No TLS on any network interface.** All coordinator WebSocket endpoints use `ws://` (plaintext). The daemon connects without TLS. Auth tokens travel in cleartext on the wire. Any attacker on the same network can sniff the token and control all dataflows.

**[C2] Daemon sends auth token as URL query parameter** (`?token=<hex>`) instead of `Authorization: Bearer` header. Query params are logged by every proxy/load balancer/access log. The coordinator already supports header-based auth — the daemon just doesn't use it.

### HIGH

- **[H1] Auth defaults to disabled.** If no `.adora-token` exists, all connections are accepted unconditionally.
- **[H2] `ADORA_ALLOW_SHELL_NODES` missing from env denylist.** Shell node gate checked via daemon env var but not in `ENV_DENYLIST`.
- **[H3] Shared memory permissions macOS/BSD — world-readable.** Permission hardening only on Linux.
- **[H4] Full dataflow descriptor in `ADORA_NODE_CONFIG` env var.** Visible via `/proc/<pid>/environ`.
- **[H5] `shellexpand::env()` expands daemon env vars into node paths.** YAML author can exfiltrate secrets.
- **[H6] No integrity check on URL-downloaded binaries.** No checksum or signature verification.
- **[H7] `NodeId` allows arbitrary strings — path traversal risk.** Node IDs flow into filesystem paths.

### MEDIUM

- No rate limiting per source IP on WebSocket endpoint
- CORS feature declared but not configured
- Downloaded binaries set to `0o764` instead of `0o700`
- `args` split on ASCII whitespace — no shell quoting support
- Error messages sent to clients include internal stack traces (`{err:?}`)
- Unix domain sockets created under `/tmp` with TOCTOU window
- `raw_sync_2` crate is an unvetted personal fork
- Auth token falls back to CWD file — spoofable

## 2. Architecture Findings

### HIGH

- **Bincode-in-JSON UTF-8 assumption.** Coordinator embeds raw bincode bytes as JSON string field, relying on UTF-8 validity.
- **Daemon event loop too large.** `binaries/daemon/src/lib.rs` exceeds 35,000 tokens.

### MEDIUM

- `async_trait` on `Connection` trait unnecessary (MSRV 1.85 supports native async fn in traits)
- Shared memory cache bounded by count (20), not bytes
- Protocol versioning absent on `ControlRequest`
- Dual logging systems (`log` + `tracing`)
- Coordinator is single-process bottleneck with `Mutex<HashMap>` for pending replies
- `adora-cli` builds as `cdylib` unconditionally
- Root `Cargo.toml` is both workspace and package

## 3. CLI UX Findings

### HIGH

- No timeout on coordinator connection loop in `adora up` — hangs forever
- Unsafe pointer aliasing in `topic echo`
- `start_coordinator` relies on fragile `argv[0]` instead of `std::env::current_exe()`
- `--version` spawns external processes on every invocation

### MEDIUM

- Memory display always "GB" even for small values
- `--follow` log mode polls at 200ms instead of inotify/kqueue
- Status messages mix stdout/stderr inconsistently
- `Build` command duplicates `CoordinatorOptions` fields
- Unicode check marks don't render on Windows
- Terminal not restored on panic in `adora top`
- Graph file collision creates numbered suffix silently

### MISSING CLI FEATURES

- `--dry-run` for destructive commands
- JSONL output not documented
- `adora system` / `adora inspect` namespaces underused
- Shell completion detection should fall back to `$SHELL`

## 4. Documentation Findings

### HIGH

- `CONTRIBUTING.md` severely outdated — references old CI jobs
- `examples/python-dataflow/README.md` describes webcam pipeline but code is numeric sender/receiver
- `docs/cli.md` has duplicate "Utility Commands" heading
- Distributed deployment docs missing `--machine-id` flag

### MEDIUM

- `docs/logging.md` says default port is `5555` — actual is `6013`
- `adora coordinator` and `adora daemon` flags not documented in cli.md
- Environment variables scattered across 5+ docs
- `docs/api-rust.md` Quick Start imports `fastrand` without listing dependency
- `_unstable_` prefix policy never explained
- `redb-backend` default status conflicts between docs

### LOW

- Several examples lack READMEs
- `--test-threads=1` requirement buried in testing guide
- WebSocket protocol docs not linked from README
- `Metadata`/`MetadataParameters` underdocumented
- `adora/timer/hz/<N>` never demonstrated
